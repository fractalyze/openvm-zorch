## One non-negotiable

- **OpenVM/SWIRL-specific only.** This repo holds the SWIRL glue (stacked PCS
  layout, query-strided Merkle structure, prismalinear RS message, byte-match
  against openvm-stark-backend). Anything scheme- or zkVM-agnostic belongs
  upstream in `zorch`, not here. If a generic block is missing, add it to
  `zorch` and depend on it — do not fork it into `openvm-zorch`.

## Byte-match

Byte-matching openvm-stark-backend `v2.0.0` (`16d60de7`) is the
correctness bar: a stage is done when it reproduces the reference prover's
bytes (canonical `u32`, no tolerances), not when a self-consistent round-trip
passes. Golden fixtures are vendored per module under `testdata/`, generated by
[`tools/fixture-gen`](tools/fixture-gen/README.md) (synthetic instance) and
[`tools/real-block-gen`](tools/real-block-gen/README.md) (real openvm guest
block) — see those READMEs for regeneration. Claims
about the reference prover cite permalinks pinned to a tag, never a branch
(branches move; a pinned link stays true). Names track the reference's
vocabulary (`l_skip`, `n_stack`, `k_whir`, "stacked", "prismalinear") so the
Python diffs cleanly against the Rust side.
